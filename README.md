# Manga Text Detector

A CNN-transformer based text detector for manga pages. The model predicts two types of heatmaps:

1. A **character heatmap** for locating text regions.
2. An **affinity heatmap** for linking nearby character/text components.

Manga text detection is challenging because text can appear in many different forms, including stylized SFX, handwritten notes, speech-bubble dialogue, and narration boxes. Instead of directly predicting bounding boxes, this project approaches the task as dense heatmap prediction. Heatmaps can represent irregular text shapes more naturally than rectangular boxes, which is useful for manga pages where text is often curved, rotated, distorted, or integrated into the artwork.

The model follows a U-Net encoder-decoder structure with a transformer bottleneck, and is trained on manga pages from One-Punch Man.


## Architecture

The architecture is characterized by three main sequences:
1. **Encoder**: It extracts increasingly abstract image features while reducing spatial resolution.
2. **Bottleneck**: This part combines dilated convolutions with a transformer encoder. The dilated convolutions expand the local receptive field, while the transformer encoder processes the compact feature map globally.
3. **Decoder**: This part upsamples the feature maps back to the original image size and uses skip connections to recover detail.

For the full layer-by-layer explanation, see [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md).

## Data generation

The training data is generated by rendering synthetic manga-style text on manga-like backgrounds, in combination with annotated and self-training pseudo-labeled images. The generator can create various text configurations, speech bubbles, backgrounds, augmentations, and target heatmaps. 

For details, see [docs/DATA.md](docs/DATA.md).

## Future Roadmap and Status
The model currently produces good predictions for regular manga text, both inside and outside speech bubbles. However, performance is still limited in some cases by character size and scaling. The synthetic text configurations do not always match the full range of text sizes found in real manga pages, and experimentation is also constrained by the available hardware.

Stylized SFX text remains one of the main open challenges. In future work, I plan to extend the data generation pipeline so that SFX-style text can be inserted more realistically during training. This should help the model learn larger, distorted, rotated, and more visually integrated text effects.

Another direction I want to explore is splitting the final prediction stage into separate character and affinity streams with a cross-attention block between them. This would allow the character and affinity representations to communicate before producing their final heatmaps, making the outputs more consistent and better correlated.

Improving the model further will likely require more real training examples. Additional annotated manga pages and self-training pseudo-labeled data should help bridge the gap between synthetic samples and real manga pages, although collecting and cleaning this data requires significant manual work.
